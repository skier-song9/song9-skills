# Notion Page Rules

When creating a Notion page, follow the format specified below.
When updating a Notion page, follow the format of the existing content.
If existing content needs to be removed, obtain permission from the user before removing it.

## Scope

These rules apply only to pages.

## Default Page Structure

### Table of Contents
A Table of Contents followed by a divider must always appear immediately after the page title.
The Table of Contents block must use a yellow background color.

### Headings
Every heading block must start with an emoji.
Use the following emoji for each heading level:
- Heading 1: ◾
- Heading 2: 🔻
- Heading 3: 🔸
Headings should be written in English whenever possible.
Technical terms must remain in their original English form.

### Descriptive Text
All explanatory text must be written using either Bulleted lists, or Numbered lists.
The preferred language for descriptions is Korean.

### Coding Elements
Simple coding elements such as (bash commands, function names, variable names, parameters) should be written as text and then formatted using "mark as code".
If the code contains two or more lines, it must be written using a code block.

### Emphasis
Important explanatory phrases should be written in bold text.

### Examples
When providing examples:
  Use a toggle block.
  The toggle block must be formatted with a green background color.

### Mathematical Expressions
If a complete mathematical expression includes inequality or equality signs, use a block equation.
If only simple mathematical symbols are used, use an inline equation block.

### Citations and Sources
Use a quote block for:
  Web search sources
  Research paper references
  Any citations or quoted text

### Warnings and Important Notes
Use a callout block when writing important cautions or warnings.

### Tables
When using tables:
  Always enable either a header row or a header column.
  Enable fit to width.
Databases must not be used.


## Tone and Voice
Use the ~이다 / ~다 declarative style. Focus on information delivery rather than politeness.
Do not include internal narration or monologue about the instructions.
Avoid statements such as:
  “I will now organize the content.”
  “I will start from here.”
  “I will read this document.”
